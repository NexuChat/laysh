# G4 accessibility and visual QA

## Automated findings

- Chrome accessibility tree: zero unnamed interactive nodes on the Arabic ask view.
- DOM integrity: zero duplicate IDs and every visible interactive target is at least 24×24 CSS px.
- Keyboard path: real Tab events reached `question` → `safe-example` → `ask-submit` in DOM order.
- Focus: the keyboard-focused primary action exposed a visible outline of at least 2 CSS px.
- Language: `lang="ar"`, document RTL, user/model text uses `dir="auto"`, and technical model text is isolated LTR.
- Localization: zero unisolated English words on the visible Arabic ask view.
- Reflow: no horizontal document overflow at 320 CSS px.
- Zoom proxy: no horizontal document overflow at a 720 CSS px viewport with a 2× page scale.
- Reduced motion: the pulse animation collapses to the reduced-motion duration.
- Recovery: answer-only, unsafe redirect, generation failure, runtime error, backend unavailable,
  cancellation, reconnecting, and still-testing states were exercised in Chrome.
- Runtime hygiene: zero unexpected console errors and zero static-resource network failures.

The accepted mobile and desktop screenshots were visually inspected for hierarchy, clipping,
RTL alignment, simulation prominence, and readable focus/action placement.

## Manual checks still required before any WCAG conformance claim

- Listen to the complete flow with an Arabic-configured screen reader such as VoiceOver or NVDA;
  verify announcement timing for the pinned answer, heartbeat, reconnect, and terminal states.
- Repeat 200% zoom using the browser menu and OS text scaling on physical mobile and desktop
  devices; DevTools emulation is a strong reflow check but is not identical to every platform.
- Inspect text and focus contrast in forced-colors/high-contrast modes and inside generated canvas
  scenes; DOM automation cannot judge the semantic contrast of every generated visualization.
- Confirm touch comfort, Arabic line breaking, and classroom projection legibility on representative
  Android/iOS devices and a projector.
- Have an Arabic-speaking learner and educator review whether status announcements and scientific
  alternatives are understandable without visual context.

No WCAG conformance claim is made at G4; the automated checks above pass, while these manual checks
remain explicit release work.
